Deployed Windows Server 2025 as a Domain Controller and configured Active Directory infrastructure

What I did:

-Created a new VM in VMware Workstation for Windows Server 2025

-Made sure preferred DNS server was set to 127.0.0.1 (localhost), because AD is dependent on DNS for operations

-Connected to LAN VMnet alongside existing pfSense and Wazuh VMs

-Promoted machine to Domain Controller
Created a new forest with the root domain lab.local
Set a DSRM recovery password

-Built a corporate OU structure under a top level Corp OU:
Departments: HR, IT, Finance, Accounting, Sales, Management, Service Accounts, Workstations, Servers
These OUs simulate a real organization's departments

Created 15 users total spread across departments
-Created departmental security groups and populated them:
Groups: HR-Users, IT-Users, Finance-Users, Accounting-Users, Sales-Users, Management-Users
Added each user to their OU and corresponding group, simulating the onboarding process for new employees

-Configured Group Policy Objects:
Password Policy: 10 character minimum, complexity enabled, 90 day expiration, 10 password history, lockout after 5 failed attempts
Screen Lock: 15 minute idle timeout with password protected screen saver
RDP Restrictions: Non-IT users can not use Remote Desktop Services
Restrict Control Panel: Non-IT Users can not access Control Panel
These GPOs together enforce a basic corporate security baseline, significantly reducing attack surface by controlling access, enforcing password standards, and limiting what standard users can do

-Performed basic helpdesk operations:
Reset a user password via right click Reset Password in ADUC
Noted that User must change password at next logon was greyed out due to minimum password age policy, but normally you would likely check this
Practiced disabling and re-enabling user accounts to simulate offboarding and security incident response

What I learned:
-A Domain Controller must point DNS to itself (127.0.0.1) after AD promotion because Active Directory is entirely dependent on DNS for authentication, replication, and locating domain services
-Computer Configuration GPOs apply to the machine regardless of who logs in, User Configuration GPOs follow the user account across machines
-Security filtering in GPMC controls which users or groups a GPO applies to, removing Authenticated Users and adding specific groups restricts the policy to only those groups
-Disabling an account is more thorough than locking it for security incidents, locked accounts can sometimes be unlocked by the user, disabled accounts require admin intervention
-Minimum password age policy prevents users from immediately reverting to an old password
-GPO best practice is one policy per GPO for easier troubleshooting and organization
